# AGENTS.md
Quick context for agents working in `nauman.im`.

## App Summary
- This repo powers `https://nauman.im`.
- It is a static site deployed from `public/` with Cloudflare Workers and Wrangler.
- There is no framework build step and no markdown-to-HTML generator.
- HTML files in `public/` are the actual site pages.
- SEO metadata is generated by `scripts/seo-optimize.mjs`.

## Key Directories
- `public/`: deploy source for the live site.
- `content/blogs/`: markdown source-of-truth for blog posts.
- `content/social/`: markdown source-of-truth for preserved social posts.
- `public/writing/index.html`: main topic hub for internal linking.
- `public/index.html`: homepage and featured writing list.
- `scripts/seo-optimize.mjs`: sitewide SEO generator.

## Canonical URL Rules
- Home: `/`
- Writing archive: `/writing/`
- Resume: `/resume/`
- `/resume.html` is a redirect and must stay non-indexable.
- Normal article route: `/<slug>/`
- Social post route: `/social/<slug>/`
- Avoid new indexable `.html` canonical URLs.

## Current Workflow
- Regenerate SEO: `node scripts/seo-optimize.mjs`
- Deploy: `pnpm exec wrangler deploy`
- Deploy config lives in `wrangler.jsonc`.

## How To Add A New Blog Post
1. Pick a stable lowercase kebab-case slug.
2. Add `content/blogs/<slug>.md`.
3. Add `public/<slug>/index.html`.
4. Put article assets in `public/<slug>/`.
5. Add the post to the right section in `public/writing/index.html`.
6. Add it to `public/index.html` if it should appear on the homepage.
7. Run `node scripts/seo-optimize.mjs`.
8. Validate before deploy.

For preserved social content use:
- `content/social/<slug>.md`
- `public/social/<slug>/index.html`

## Article Page Expectations
- Exactly one `<h1>`.
- Use `<main>` for the main content.
- Use `<article>` when the layout is article-like.
- Keep real text in HTML, not JS-rendered content.
- Include a visible publish date when possible.
- Include a home or archive link.
- Add meaningful internal links where natural.

## SEO Rules
- Do not manually maintain SEO blocks in `<head>` unless there is a good reason.
- Let `scripts/seo-optimize.mjs` own titles, descriptions, canonicals, Open Graph, Twitter, JSON-LD, sitemap, RSS, and robots.
- Do not add `meta keywords`.
- Prefer short, descriptive, non-clickbait titles.
- Use one canonical URL per page.
- Helpful content matters more than keyword stuffing.

## What The SEO Script Does
- Injects metadata and structured data.
- Adds favicon and Apple touch icon links.
- Generates `sitemap.xml`, `rss.xml`, and `robots.txt`.
- Adds related-article blocks to article pages.
- Adds image `width`, `height`, `decoding`, `loading`, and `fetchpriority` where possible.
- Uses source files and git history for `dateModified`.

## When To Update Script Overrides
- `TITLE_OVERRIDES`: when the natural title is too long or weak.
- `DESCRIPTION_OVERRIDES`: when the generated description is inaccurate.
- `DATE_OVERRIDES`: when exact publish date matters and detection is unreliable.
- `IMAGE_OVERRIDES`: when the first page image is not the right OG image.
- `ROUTE_SOURCE_OVERRIDES`: when a route maps to a non-standard content source.

## Writing Archive Rules
- `public/writing/index.html` is the main topical hub.
- Put each new article in the most relevant topic section.
- Add a new section only when a real topic cluster exists.
- Keep time-sensitive older posts framed as historical when needed.

## Homepage Rules
- `public/index.html` should stay selective and concise.
- If a post belongs on the homepage, keep the list in sensible editorial order, usually newest first.
- Do not add every article to new homepage sections.

## Images And Assets
- Always write useful `alt` text.
- Prefer local assets over remote hotlinks.
- Keep article assets inside the article folder.
- If a hero image should be the social share image, make sure the script can detect it or add an `IMAGE_OVERRIDES` entry.

## Redirects And Canonicals
- Route changes belong in `public/_redirects`.
- Use `301` for old paths that should resolve to a new canonical path.
- Do not leave duplicate indexable routes.

## Validation Before Commit
- Run `node scripts/seo-optimize.mjs`.
- Check the new page has one `<h1>`.
- Check there are no broken local asset paths.
- Check the page appears in `public/sitemap.xml`.
- Check the page appears in `public/rss.xml` if it is an article.
- Check `public/writing/index.html` and `public/index.html` were updated when needed.

## Validation Before Deploy
- Open the page in a browser.
- Confirm the canonical URL is correct.
- Confirm there are no console errors from missing local files.
- Confirm internal links and topic placement make sense.

## Files To Treat Carefully
- `scripts/seo-optimize.mjs`
- `public/_redirects`
- `public/index.html`
- `public/writing/index.html`

## Preferred Agent Behavior
- Create the content page first.
- Update archive and homepage placement second.
- Regenerate SEO third.
- Validate before deploy.
- If you introduce a new route type or workflow, update this file.
